# 问题排查指南

## 🔍 问题分析

如果您无法打开界面，请尝试以下几种方法：

### 方法1：直接打开HTML文件（最简单）

**步骤：**
1. 打开文件资源管理器
2. 进入 `D:\opencode` 目录
3. 找到 `index.html` 文件
4. **双击打开**

✅ **推荐使用此方法**，可以直接查看界面效果！

### 方法2：使用批处理文件启动服务器

**步骤：**
1. 在 `D:\opencode` 目录中找到 `start.bat` 文件
2. 双击运行
3. 等待看到服务器启动信息
4. 在浏览器中打开：http://localhost:3000

### 方法3：手动启动服务器

**步骤：**
1. 打开命令提示符（CMD）
2. 输入以下命令：
   ```
   cd D:\opencode
   npm install
   node server.js
   ```
3. 在浏览器中打开：http://localhost:3000

## 📱 访问地址

- **主页**: http://localhost:3000 或 http://localhost:3000/index.html
- **直接文件**: file:///D:/opencode/index.html

## ⚠️ 注意事项

1. **防火墙问题**
   - 如果被防火墙阻止，请允许Node.js访问网络

2. **端口占用**
   - 如果3000端口被占用，可以修改 `.env` 文件中的 `PORT` 值

3. **依赖未安装**
   - 确保已运行 `npm install`

4. **Node.js版本**
   - 确保安装了 Node.js 14 或更高版本

## 🎯 推荐的访问方式

### 方式A：直接打开HTML（无需服务器）
```
双击 D:\opencode\index.html
```
优点：简单快速，无需配置
缺点：无法使用AI视频生成功能（仅查看界面）

### 方式B：使用服务器（完整功能）
```
双击 D:\opencode\start.bat
然后访问 http://localhost:3000
```
优点：所有功能可用
缺点：需要配置API密钥

## 🔧 故障排除

### 问题1：双击HTML文件打不开
- 确保安装了浏览器（Chrome、Edge等）
- 右键 → 打开方式 → 选择浏览器

### 问题2：服务器启动失败
- 检查是否安装了 Node.js：`node --version`
- 重新安装依赖：`npm install`
- 查看错误信息并解决

### 问题3：浏览器显示"无法连接"
- 检查服务器是否正在运行
- 尝试使用不同的端口
- 直接打开HTML文件（方式A）

## 💡 最佳实践

**对于快速预览：**
→ 直接双击 `index.html` 文件

**对于完整功能：**
→ 双击 `start.bat`，配置API密钥后使用

## 📞 获取帮助

如果问题仍未解决：
1. 检查错误日志
2. 确认所有文件都存在
3. 尝试重新安装依赖
4. 使用直接打开HTML文件的方式
